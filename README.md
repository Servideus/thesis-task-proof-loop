# thesis-task-proof-loop

Репозиторий со skill для Codex-подобных агентов, который помогает вести **академическую работу над текстом** через контролируемый цикл: планирование, фиксация источников, драфт по утверждённым материалам, проверка и точечная доработка.

## Зачем это нужно

Этот репозиторий содержит правила, роли, шаблоны и примеры запуска для дисциплинированного цикла исследовательского письма:

**заморозка требований -> фиксация утверждённых источников -> драфт только по утверждённым материалам -> карта утверждений к источникам -> свежая проверка -> точечные правки -> повторная проверка**

Цель — не генерировать красивую ерунду, а поддерживать академическое письмо с трассируемостью, дисциплиной по источникам и внятной проверкой.

## Чем skill должен помогать

- фиксировать объём и границы главы или раздела;
- организовывать утверждённые источники;
- писать драфт по требованиям пользователя, заметкам и утверждённым материалам;
- связывать содержательные утверждения с источниками;
- проверять структуру, логику и дисциплину цитирования;
- вносить точечные правки вместо бессмысленного переписывания всего текста.

## Чего skill никогда не должен делать

- выдумывать ссылки и библиографию;
- делать вид, что прочитал недоступный источник;
- придумывать цитаты, страницы, DOI или URL;
- незаметно уплывать от утверждённого плана.

## Основные файлы

- `SKILL.md` — главные рабочие инструкции для агента.
- `docs/ROLES.md` — роли и зоны ответственности.
- `docs/WORKFLOW.md` — полный цикл работы и условия остановки.
- `docs/QUALITY_GATES.md` — критерии качества и чеклист проверки.
- `docs/USAGE_PROMPTS.md` — промпты для запуска skill в Codex.
- `templates/` — шаблоны рабочих артефактов.

## Ожидаемые рабочие артефакты в целевом репозитории

```text
.agents/tasks/<TASK_ID>/
  spec.md
  requirements.md
  sources_registry.md
  outline.md
  draft.md
  claim_source_map.csv
  evidence.md
  problems.md
  revision_log.md
  verdict.json
```

## Рекомендуемое использование

Подключай этот репозиторий как источник skill для Codex. Сам skill должен запускаться внутри рабочего репозитория с дипломом или другой академической работой и создавать файлы в `.agents/tasks/<TASK_ID>/`.

Дополнительные примеры есть в `docs/USAGE_PROMPTS.md`.

## Как запускать этот skill в Codex

Ниже — готовые примеры промптов, которые можно вставлять в Codex, когда skill уже доступен.

### 1. Базовый промпт запуска

Используй его, когда нужно инициализировать или продолжить задачу по работе над дипломом в текущем репозитории.

```text
Создай субагентов. Используй $thesis-task-proof-loop, чтобы продолжить задачу, описанную ниже, в этом репозитории.
Если подходящая задача уже существует, переиспользуй её; если нет — сначала инициализируй.
Строго следуй skill:
- заморозь задачу;
- зафиксируй утверждённые источники;
- построй или подтверди план;
- пиши драфт только по утверждённым материалам;
- свяжи содержательные утверждения с источниками;
- проведи проверку в свежем контексте;
- вноси только точечные правки;
- если база источников недостаточна, останавливайся со статусом FAIL.

Задача:
Написать или доработать главу 1 дипломной работы.

Требования:
- язык: русский;
- сохранить утверждённую структуру;
- не выдумывать ссылки и библиографические данные;
- все рабочие артефакты хранить в .agents/tasks/<TASK_ID>/.
```

### 2. Промпт для написания конкретного раздела

Используй его, когда план и источники уже лежат в репозитории.

```text
Создай субагентов. Используй $thesis-task-proof-loop в этом репозитории.
Задача: подготовить раздел 1.2 дипломной работы по утверждённому плану, заметкам и источникам, которые уже есть в репозитории.

Требования:
- писать на русском академическом языке;
- не использовать неутверждённые источники;
- не выдумывать цитаты, страницы, DOI и URL;
- создать или обновить draft.md, claim_source_map.csv, evidence.md и verdict.json;
- если источников недостаточно, остановиться и чётко объяснить, чего именно не хватает.
```

### 3. Промпт только для проверки

Используй его, когда раздел уже написан и нужно проверить его, а не переписывать с нуля.

```text
Используй $thesis-task-proof-loop в режиме проверки для существующей папки задачи.
Не переписывай раздел широко.
Проверь quality gates, дисциплину по источникам и трассируемость, обнови evidence.md, запиши verdict.json и перечисли конкретные дефекты в problems.md.
```

### 4. Промпт для точечных правок

Используй его, когда `problems.md` уже существует и нужно внести только узкие исправления.

```text
Используй $thesis-task-proof-loop в режиме исправления для существующей папки задачи.
Прочитай problems.md и внеси только точечные исправления.
Не делай полный рерайт.
Обнови revision_log.md и снова запусти проверку.
```

### 5. Промпт под диплом о цифровизации сельского хозяйства России и Китая

```text
Создай субагентов. Используй $thesis-task-proof-loop, чтобы продолжить задачу, описанную ниже, в этом репозитории.
Если подходящая задача уже существует, переиспользуй её; если нет — сначала инициализируй.

Задача:
Подготовить или доработать главу дипломной работы о цифровизации сельского хозяйства и её экономических последствиях на примере России и Китая.

Требования:
- язык: русский;
- жанр: академический текст для ВКР или магистерской работы;
- если структура главы уже утверждена, сохранить её;
- опираться только на утверждённые источники, которые есть в репозитории или явно одобрены пользователем;
- не выдумывать ссылки, цитаты, статистику и библиографические детали;
- создавать и поддерживать .agents/tasks/<TASK_ID>/spec.md, requirements.md, sources_registry.md, outline.md, draft.md, claim_source_map.csv, evidence.md, problems.md, revision_log.md и verdict.json;
- если база источников слишком слабая для сильного утверждения, либо ослабить формулировку, либо остановиться со статусом FAIL.
```

## Предлагаемый порядок работы для пользователя

1. Положить в рабочий репозиторий план, заметки, черновики и утверждённые источники.
2. Убедиться, что Codex видит этот skill.
3. Вставить в Codex один из промптов выше.
4. Дать skill создать папку задачи и рабочие артефакты.
5. Проверять `sources_registry.md`, `draft.md`, `problems.md` и `verdict.json` вместо ручного перебрасывания длинных сообщений между чатами.

## Практическое замечание

Этот skill работает лучше всего, когда в репозитории уже есть:
- план диплома или план главы;
- заметки и выписки;
- файлы утверждённых источников или хотя бы корректные библиографические записи;
- требования к оформлению.

Если этих входных данных нет, skill должен не делать вид, что всё понял, а замедляться, фиксировать нехватку структуры и требовать недостающие опоры.
