# Промпты для запуска

## Базовый промпт

```text
Создай субагентов. Используй $thesis-task-proof-loop, чтобы продолжить задачу, описанную ниже, в этом репозитории.
Если подходящая задача уже существует, переиспользуй её; если нет — сначала инициализируй.
Строго следуй skill:
- заморозь задачу;
- зафиксируй утверждённые источники;
- построй или подтверди план;
- пиши драфт только по утверждённым материалам;
- свяжи содержательные утверждения с источниками;
- перед сменой роли или новой крупной фазой обновляй артефакты задачи и handoff.md;
- запускай проверку только в свежем контексте;
- вноси только точечные правки;
- если базы источников недостаточно, остановись со статусом FAIL.
```

## Промпт для написания раздела по уже утверждённым материалам

```text
Создай субагентов. Используй $thesis-task-proof-loop в этом репозитории.
Задача: подготовить структурированный раздел по утверждённому плану, заметкам и источникам, которые уже лежат в репозитории.

Требования:
- не использовать неутверждённые источники;
- не выдумывать библиографические данные;
- все рабочие артефакты хранить в .agents/tasks/<TASK_ID>/;
- создать draft.md, claim_source_map.csv, handoff.md, evidence.md и verdict.json;
- если происходит передача задачи другой роли, сначала обновить handoff.md, потом продолжить работу в свежем контексте.
```

## Промпт только для проверки

```text
Используй $thesis-task-proof-loop в режиме проверки для существующей папки задачи.
Запускай проверку в свежем контексте.
Перед началом прочитай spec.md, requirements.md, sources_registry.md, outline.md, draft.md, claim_source_map.csv, handoff.md, evidence.md, problems.md и revision_log.md, если они уже существуют.
Не делай широкий рерайт раздела.
Проверь quality gates, обнови evidence.md, запиши verdict.json и перечисли конкретные дефекты в problems.md.
```

## Промпт для точечных правок

```text
Используй $thesis-task-proof-loop в режиме исправления для существующей папки задачи.
Перед началом прочитай handoff.md и problems.md.
Прочитай problems.md и внеси только точечные исправления.
Не делай полный рерайт.
После правок обнови revision_log.md и handoff.md, затем снова запусти проверку в свежем контексте.
```
